# Notebook cleanup checklist

Before you treat this project as portfolio-grade, fix these points:

- remove repeated exploratory cells that do the same thing twice
- collapse or delete noisy print-heavy debugging outputs
- rename sections so they sound professional, not assignment-like
- make file naming consistent everywhere
- keep one clear final training path
- export final charts into `assets/`
- add one markdown cell at the top:
  - problem
  - approach
  - final metrics
  - model file saved
- remove student ID references from visible filenames if possible
- keep the notebook as research evidence, not your only implementation
