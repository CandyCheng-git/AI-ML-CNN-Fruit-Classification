# Data README

The original project used separate `train/` and `test/` image folders with 14 fruit classes.

The raw dataset is not included in this portfolio bundle.

## Why

- keeps the repository lightweight
- avoids pushing thousands of image files unnecessarily
- keeps focus on code, workflow, and model outcome
- reduces the chance of violating dataset redistribution rules

## If you want to add sample data

A good compromise is:

- create `sample_data/`
- add 1 to 3 images per class only
- use those images for demo inference screenshots

That gives recruiters something visual without turning the repository into a storage dump.
